内容电商相关

发表于 2018-12-17

一.目前的内容电商

说起内容电商，目前主流的内容电商主要分为两类：

  • 以精选内容为主，通过社区的形式引导一部分用户产生内容，普通用户对于一部分商品知识有获取的需要，进一步引导转化。比如小红书。
  • 以IP为主，通过内容让用户对IP产生信任，满足用户获取知识的需要，基于对IP的信任完成转。比如年糕妈妈。

淘宝从很早之前就开始做内容电商，最早的淘女郎，现在的微淘，都是为了满足用户逛的需求。其理念是，用户把时间花在这里了，总会买东西的。如果非要归类，我会把他归为第二类，用户基于对商家或品牌的信任，获取内容和购买商品。

对拼多多的十次思考

发表于 2018-10-29

记得第一次了解拼多多是在16年的7月，而如今其在纳斯达克上市也已经过去两个月了，这中间，不过两年而已。两年来，拼多多一直饱受争议，有批判其卖假货的，产品没有下限的，也有抄袭他流量获取方法的，有褒扬他供应链效率的。而我恰恰对其保持了持续的关注，期间产生了一些思考，分享给大家。

一.模式新颖

第一次认识拼多多的时候，那会刚毕业不久，对电商的了解也实在有限。仅仅知道拼多多在微信里面利用拼团的方式，将拉新的成本让利给了用户，最终实现了以较低的成本实现了获取新客。对拼多多的理解停留在创意新颖。

二.货品瓶颈

第二次思考拼多多的时候，已经对电商有了一定理解，认为货品是一个电商的核心。而拼多多留给我的标签是“价格便宜”、“质量差”，靠罚商家的钱盈利。我认为货品质量差的话，早晚会制约其发展，货品会成为增长的瓶颈，拼多多会陷入到便宜的东西质量不好，质量好的东西不便宜，价格贵的东西对用户没有吸引力，质量差的东西会伤害用户的怪圈中去。靠罚商家的钱盈利的商业模式不可持续，当时是不看好拼多多的。

广告思维与零售思维

发表于 2018-09-30

思考

  • 广告思维是在一次pv下，通过各环节的优化提升转化率，获得产出。所以广告商品更关注卖点（适合营销）和利润（获取pv）
  • 零售思维是在一次购物中给用户留下好的印象（确定性）。所以电商商品更关注品类（复购场景）和体验（复购可能性，包括价格）
  • 电商产品要在广告思维和零售思维下取得平衡点

case

  • 广告爆品在取得较好的转化后，没有产生较好的复购，流量依赖性大。用户看到商品了觉得不错会购买，但是不会主动想起来要来买东西
  • 拼多多用快消品不断地触达用户的过程中，给用户传递了价格便宜的印象，用户（拼多多的用户）在日常生活中想起要买东西的时候，先去拼多多看下

特征值检验

发表于 2017-06-16

特征检验

卡方检验

特征/结果 R 非R 总计
A a b a+b
非A c d c+d
总计 a+c b+d a+b+c+d

令样本数N=a+b+c+d

假设A与R无关，即A＝非A

则R的概率应该是(a+c)/N

那么得到期望：

特征/结果 R 非R 总计
A (a+c)/N*(a+b) (b+d)/N*(a+b) a+b
非A (a+c)/N*(c+d) (b+d)/N*(c+d) c+d
总计 a+c b+d a+b+c+d

根据卡方检验的公式: 

D = ( ∑(E1-E0)² )/E0，E1是实际值，E0是理论值（期望值）

D越大（A,R）越相关，D越小(A,R)越不相关

KA商家运营

发表于 2017-04-06

B2C的平台(bussiness to customer)，顾名思义，核心业务是通过平台把商家的货(B端)直接卖给消费者(C端)。所以这一类的企业，不管是淘宝，还是其他类似的平台，商家运营的工作就显得十分重要，因为对于消费者来说，平台的核心价值就是货，而货都是商家提供的。此外，商家运营，因为在划分组别的时候大都按照品类规划，所以很多地方也会叫做类目运营，行业运营，品类运营，但是核心工作其实都是商家运营。

就像很多企业会圈定自己的核心用户一样，受限于人员精力以及运营成本，电商运营的过程中也对自己的平台的商家去圈定核心商家进行重点运营，主要工作有如下几点:

  • KA商家选择，建立KA商家池
  • KA商家运营（合作，推广，培养）
  • KA商家迭代

写在数据分析最前面

发表于 2017-04-05

往往我们谈到数据分析，或者说想要从事数据分析工作，自然而然地，我们首先会想到R,Python等编程语言，亦或是决策树，神经网络等分析算法。在大数据概念火热的年代，”啤酒与尿布”的神话给数据分析披上了一层高贵的面纱。然而在我看来，数据分析并非如此高贵冷艳，人人都可以成为数据分析师，因为我们追寻本质，数据分析无非就是解决问题的一个途径罢了。而最终影响你的高度的，可能是你对这个行业的理解深度，也就是你对这个行业的问题的理解深度。所以写在数据分析最前面，我认为我们始终要聚焦到以下几点：

  • 面向什么用户什么场景，遇到了什么问题
  • 需要用到什么样的数据
  • 对数据进行了怎样的处理和分析
  • 通过数据得到了什么结论
  • 得到的结论带来了什么价值
